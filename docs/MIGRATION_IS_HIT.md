# Міграція: поле is_hit для товарів (Хіти продажів)

## Як у проекті змінюється схема БД

У проекті **не використовується Alembic**. Структура БД оновлюється так:

1. **При старті сервера** викликається `fix_db_schema()` у `main.py`, яка виконує `CREATE TABLE IF NOT EXISTS` та `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` для нових полів.
2. Поле `is_hit` уже додано в `fix_db_schema()`: при першому запуску FastAPI після оновлення коду колонка `is_hit` буде створена автоматично, **існуючі товари не видаляються**.

---

## Якщо потрібно виконати міграцію вручну (консоль БД)

Підключіться до PostgreSQL (наприклад, `psql` або клієнт за вашим `DATABASE_URL`) і виконайте **один** із варіантів.

### Варіант 1: Безпечний (рекомендовано)

Колонка додасться тільки якщо її ще немає; дані не зачіпаються:

```sql
ALTER TABLE products ADD COLUMN IF NOT EXISTS is_hit BOOLEAN DEFAULT FALSE;
```

### Варіант 2: Класичний ALTER (якщо IF NOT EXISTS не підтримується)

Якщо ваша версія PostgreSQL не підтримує `IF NOT EXISTS` для `ADD COLUMN`:

```sql
ALTER TABLE products ADD COLUMN is_hit BOOLEAN DEFAULT FALSE;
```

Якщо колонка вже існує, цей запит поверне помилку — тоді нічого робити не потрібно.

## Після міграції

- Усі існуючі рядки отримають `is_hit = FALSE`.
- Керувати «хітами» можна через:
  - **Адмінка:** сторінка `/admin` → вкладка «Товары» → редагування товару → чекбокс «Хіт продажу»; у таблиці товарів колонка «Хіт».
  - **API:** при створенні/редагуванні товара передавати в тілі `"is_hit": true` або `false`.
